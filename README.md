## 이게 뭔가요?

AI와 자연어(말)로 코딩하는 "바이브코딩"을 시작하려면 PC에 여러 개발 도구를 미리 설치해야 합니다.  
하나씩 찾아 설치하면 처음엔 30분~1시간이 걸리고 실수도 많습니다.

**`dev-one-click-setting-kit.bat`** 파일 하나만 실행하면:

- 필요한 도구를 자동으로 찾아 설치
- 이미 설치된 도구는 건너뛰거나 업그레이드
- 설치 완료 후 결과 리포트 자동 저장

> 이 파일은 **Windows 전용**입니다. macOS/Linux에서는 작동하지 않습니다.

## 사용 방법 — 완전 초보자용

### 준비물

- Windows 10 (21H2 이상) 또는 Windows 11
- 인터넷 연결
- `dev-one-click-setting-kit.bat` 파일

### 실행 순서

#### 1단계 — 파일 받기

이 페이지 오른쪽 위 초록색 **`<> Code`** 버튼 클릭  
→ **`Download ZIP`** 클릭  
→ 다운로드된 ZIP 파일 압축 해제

#### 2단계 — 관리자 권한으로 실행

`dev-one-click-setting-kit.bat` 파일 위에서  
마우스 오른쪽 클릭 → **"관리자 권한으로 실행"** 선택

> ⚠️ 관리자 권한 없이 실행하면 일부 도구가 설치되지 않을 수 있습니다.

#### 3단계 — 메뉴 선택

검은 화면(터미널)에 메뉴가 나타납니다.  
원하는 번호를 입력하고 **Enter** 키를 누르세요.

```
============================================================
   바이브코더 환경 키트
============================================================

 [1] 초보자 설치    ← 처음이라면 여기서 시작하세요
 [2] 중급 설치
 [3] 고급 설치
 [4] 새로운 도구
 [5] 개별 선택 설치
 [6] 업데이트
 [7] 제거 (개별)
 [8] 전체 제거
 [9] 설치 확인
 [0] 종료
```

#### 4단계 — 이미 설치된 도구 처리

설치 시작 전, 이미 설치된 도구가 있으면 처리 방법을 묻습니다.

```
이미 설치된 도구 처리 방법:
  [1] 건너뜀    (현재 버전 유지)
  [2] 업그레이드 (최신 버전으로)
  [3] 제거       (삭제만)
  선택 (기본값=1): 
```

> Enter 키만 눌러도 자동으로 **[1] 건너뜀**이 선택됩니다.

#### 5단계 — 완료 확인

설치가 끝나면 같은 폴더에 결과 파일이 자동 저장됩니다.

| 파일 | 내용 |
|------|------|
| `install-report-날짜.txt` | 성공/실패/건너뜀 요약 |
| `install-log-날짜.txt` | 상세 설치 기록 |

## 설치 레벨 안내

| 레벨 | 추천 대상 | 누적 도구 수 |
|------|----------|------------|
| **[1] 초보자** | AI 바이브코딩 처음 시작하는 분 | 5개 |
| **[2] 중급** | 초보자 레벨 완료 후 더 필요한 분 | 10개 |
| **[3] 고급** | 다양한 언어로 개발하는 분 | 14개 |
| **[4] 새로운** | 특수 언어(Ruby, PHP) 필요한 분 | 16개 |

> 처음이라면 **[1] 초보자** 레벨만으로 바이브코딩 시작에 충분합니다.

## 주요 기능

| 기능 | 설명 |
|------|------|
| 원클릭 자동 설치 | Windows 기본 패키지 관리자(winget)로 자동 처리 |
| 이미 설치된 도구 감지 | 건너뜀 / 업그레이드 / 제거 선택 가능 |
| 자동 재시도 | 첫 시도 실패 시 1회 자동 재시도 |
| 이식성 보장 | BAT 파일 위치나 이름을 바꿔도 정상 작동 |
| 설치 리포트 자동 저장 | 성공/실패/건너뜀 항목을 텍스트 파일로 저장 |
| PATH 경로 확인 | 설치 후 환경 변수 등록 여부 자동 점검 |
| CLI 버전 치트시트 | 완료 후 주요 명령어 버전 확인 안내 출력 |
| 개별 선택 설치 | [1]~[26] 중 원하는 도구만 골라서 설치 |
| 업데이트 메뉴 | npm 패키지 일괄 최신화 |
| 제거 메뉴 | 개별 또는 전체 제거 지원 |

## 자주 묻는 질문

**Q. 관리자 권한으로 실행해도 설치가 안 되면?**

Windows Defender나 백신이 BAT 파일 실행을 막을 수 있습니다.  
백신 설정에서 해당 파일을 허용 목록에 추가하거나, 잠시 실시간 보호를 끄고 실행해 보세요.

---

**Q. 설치 중 창이 갑자기 닫히면?**

같은 폴더에 생성된 `install-log-날짜.txt` 파일을 열어 마지막 줄을 확인하세요.  
오류 내용이 기록되어 있습니다.

---

**Q. "winget을 찾을 수 없습니다" 오류가 나오면?**

Microsoft Store에서 **"앱 설치 관리자"** 를 검색해 설치하세요.  
또는 Windows 설정 → 앱 → 선택적 기능 → 앱 설치 관리자를 확인하세요.

---

**Q. 특정 도구만 설치하고 싶으면?**

메인 메뉴에서 **[5] 개별 선택 설치** 를 선택하면 도구를 하나씩 고를 수 있습니다.

---

**Q. 이미 설치된 도구를 최신 버전으로 올리고 싶으면?**

메인 메뉴 **[6] 업데이트** 를 선택하거나,  
레벨 설치 시 처리 방법에서 **[2] 업그레이드** 를 선택하세요.

---

**Q. 다시 실행해도 괜찮나요?**

네. 이미 설치된 환경에서 다시 실행해도 기본값 **건너뜀** 처리로 안전합니다.

## 운영 시 주의사항

- **Windows 전용** — macOS/Linux에서는 작동하지 않습니다.
- **인터넷 연결 필수** — winget이 인터넷에서 패키지를 다운로드합니다.
- **보안 정책 제한** — 일부 기업 PC는 winget 사용이 정책으로 막혀 있을 수 있습니다.
- **로그 파일 공유 주의** — `install-log-*.txt`에는 민감 정보가 없지만, 다른 사람과 공유 시 확인 후 전달하세요.
- **재실행 안전** — 이미 설치 완료된 PC에서 다시 실행해도 기존 설정을 건드리지 않습니다.
