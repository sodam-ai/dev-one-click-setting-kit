## 1. 제품 개요

### 한 줄 요약
AI 바이브코딩을 처음 시작하는 사람이 Windows PC에서 개발 환경을 원클릭으로 세팅하고 즉시 사용 가능한 상태로 만들어 주는 BAT 파일 배포 도구

### 핵심 가치
- **원클릭**: DEV-KIT.bat 더블클릭 하나로 자동 처리
- **4단계 누적**: 왕초보 → 중급 → 고급 → 올인원 (상위 단계는 하위 포함)
- **사용 가능 상태**: 설치만 아니라 Git 설정·pip 업그레이드 등 즉시 쓸 수 있는 상태까지
- **배포용**: 강사 → 수강생 파일 하나 전달로 완결
- **리포트**: 설치 완료 후 강사가 수강생 환경 파악 가능한 리포트 파일 자동 생성

### 해결하는 문제
- AI 코딩 입문자가 각 도구를 따로 검색해서 설치하는 과정이 복잡하고 오류가 잦음
- Git 설치 후 이메일 설정 없어 첫 커밋부터 실패하는 문제
- VS Code만 설치하고 확장 없어 에디터 세팅에 시간 낭비
- Ollama 설치 후 모델 없어 "왜 안 되지?" 혼란
- 강사가 수강생 30명의 설치 상태를 파악할 방법 없음

## 2. 시스템 요구사항

### 최소 환경
| 항목 | 최소 | 권장 |
|------|------|------|
| Windows 버전 | Windows 10 21H2 (빌드 19044) 이상 | Windows 11 |
| 디스크 여유 공간 | 왕초보: 3GB / 전체: 15GB | 20GB 이상 |
| 네트워크 | 전체 설치 시 약 4~6GB 다운로드 | 안정적인 유선 권장 |
| 메모리 | 4GB | 8GB 이상 |
| winget 버전 | 1.4 이상 | 최신 버전 |

### Windows 버전별 호환성
| 버전 | 지원 | 비고 |
|------|------|------|
| Windows 11 | ✅ 완전 지원 | 권장. Windows Terminal 기본 내장 |
| Windows 10 21H2+ | ✅ 완전 지원 | winget 안정 버전 |
| Windows 10 21H1 이하 | ⚠️ 부분 지원 | winget 수동 설치 필요 |
| Windows 10 1803 미만 | ❌ 미지원 | winget 없음 |

- **목표**: 오류 없이 환경 세팅 완료 + 즉시 사용 가능 상태

### 보조 사용자
- **누구**: 바이브코딩 강사
- **목표**: 수강생 개별 세팅 지원 최소화 + 설치 리포트로 환경 파악

## 6. 사용자 흐름

### 정상 흐름
```
DEV-KIT.bat 더블클릭
  → 사전 체크 (winget source update / 디스크 / 인터넷 / winget 버전 / 충돌)
  → 메뉴 표시
  → 단계 선택 (왕초보/중급/고급/올인원)
  → 의존성 순서대로 설치 ([n/N] 진행 카운터 표시)
  → Post-install 작업 (Git 설정 안내, pip 업그레이드 등)
  → 설치 리포트 파일 2종 생성
  → 설치된 도구 PATH 자동 검증 출력
  → 완료 메시지 + "새 터미널 열기" 안내
```

### 오류 분기 흐름
```
winget 없음
  → 설치 URL + Microsoft Store 안내 → 종료

Windows 10 21H1 이하 감지
  → winget 수동 설치 URL 안내 → 종료

디스크 부족
  → "X GB 필요, 현재 Y GB 여유" 표시 → 계속할지 선택

기존 Node.js 감지
  → "기존 버전 발견. 건너뜀" 처리 후 계속

네트워크 오류 (설치 중)
  → 5초 후 1회 자동 재시도 → 재시도 실패 시 "건너뜀"
  → 다음 도구 계속 → 완료 후 실패 목록 표시
```

## 7. Post-install 작업 요구사항 ★NEW

> 설치만으로는 부족. "즉시 사용 가능 상태"로 만들기 위한 추가 작업.

| 도구 | 작업 | 방식 | 확정 |
|------|------|------|------|
| Git | `git config --global user.name` / `user.email` | 안내 텍스트만 출력 (대화형 입력 금지 — 개인정보 원칙) | ✅ 확정 |
| Git | `git config --global core.autocrlf true` | 자동 실행 (Windows 줄바꿈 표준) | ✅ 확정 |
| Python | `pip install --upgrade pip` | 자동 실행 | ✅ 확정 |
| Node.js | `npm config set fund false` | 자동 실행 (npm 광고 메시지 제거) | ✅ 확정 |
| Ollama | `ollama pull llama3.2` (2GB 경고 포함) | 안내만 — 자동 다운로드 금지 (2GB 부담) | ✅ 확정 |
| VS Code | Python 확장, Prettier 설치 URL 안내 | URL 제공만 (확장 자동 설치 금지 — 취향 강요) | ✅ 확정 |
| Rust | `rustup update stable` | 자동 실행 | ✅ 확정 |
| 공통 | 설치된 도구 where 명령으로 PATH 검증 출력 | 자동 실행 | ✅ 확정 |

## 8. 설치 리포트 요구사항 ★NEW (2종 분리)

### install-report-[날짜].txt — 강사용 요약
```
파일명: install-report-20260420.txt
생성 위치: DEV-KIT.bat 실행 폴더

포함 내용:
  - 설치 날짜/시간
  - 선택한 레벨 (왕초보/중급/고급/올인원)
  - 설치 성공 도구 목록 + 버전
  - 건너뜀/실패 도구 목록
  - 소요 시간 (분)
  - PC 이름 (COMPUTERNAME)

목적: 강사가 수강생 환경 파악에 활용 (이메일/공유 폴더로 전달 안내)
주의: PC 이름 외 개인정보 포함 금지 (사용자 이름, 홈 경로 등 제외)
```

### install-log-[날짜].txt — 디버깅용 상세
```
파일명: install-log-20260420.txt
생성 위치: DEV-KIT.bat 실행 폴더

포함 내용:
  - 각 도구 설치 시작/완료 시각
  - winget errorlevel 값
  - 실패 시 winget 오류 코드
  - 사전 체크 결과 (디스크/인터넷/winget 버전)
  - 재시도 여부 기록

목적: 수강생이 오류 스크린샷 대신 이 파일을 강사에게 전달하여 원격 디버깅
주의: 사용자 홈 경로 등 민감 경로 포함 주의 — 경로 대신 "설치 성공/실패" 코드만 기록
```

## 9. 사전 체크 요구사항 ★NEW

```
설치 시작 전 자동 확인 항목:
  1. Windows 버전 확인 → 10 21H1 이하면 winget 수동 설치 URL 안내 후 종료
  2. winget 1.4+ 여부 → 없으면 Microsoft Store / 설치 URL 안내 후 종료
  3. winget source update 실행 → 실패해도 경고만, 강제 종료 금지
  4. 디스크 여유 공간 → 선택 레벨 최소 공간 미달 시 경고 (계속 여부 선택)
  5. 인터넷 연결 → 없으면 오프라인 안내
  6. 기존 Node.js 버전 감지 → 충돌 가능성 안내
  7. 기존 Python 버전 감지 → 충돌 가능성 안내
```

## 10. 성공 기준

| 기준 | 측정 방법 | 목표 |
|------|----------|------|
| 왕초보 설치 성공 | `git --version && python --version && node --version` 모두 출력 | 100% |
| 재실행 안정성 | 이미 설치된 환경 재실행 시 오류 없이 완료 | 100% |
| 수강생 독립 성공률 | 강사 도움 없이 왕초보 설치 완료 | 90% 이상 |
| 설치 시간 | 왕초보 기준 | 10분 이내 |
| Post-install | Git 설정 안내 확인 후 첫 커밋 성공 | 90% 이상 |

## 11. 보안 설계 원칙

1. **관리자 권한 불필요** — winget 사용자 범위 설치. 기업/학교 PC 대응
2. **winget 소스만 신뢰** — `--source winget` 명시. Microsoft 서명 검증 내장
3. **사용자 입력 최소화** — 메뉴 번호만 입력. 임의 명령어 실행 경로 없음
4. **Git 설정 수동 처리** — 이메일/이름은 자동 저장 금지. 안내만 제공

## 12. 안 만드는 것 (Out of Scope)

- GUI 인터페이스(.exe) — Phase 3 검토
- Mac/Linux 지원 — BAT은 Windows 전용
- WSL2 자동 설치 — 별도 프로젝트(WSL 원클릭 키트) 참조
- 오프라인 설치 패키지 — Phase 3 검토
